Discoverysysteme
Solr hat nur Admin-Oberfläche und keine Webseite. Ist mit Elasticsearch quasi ein "Industriestandard". Diese beiden OpenSource Suchmaschinen sind die meistverwendeten. 
Üblicherweise sollte vor Import der Daten in einem Schema festegelegt werden welche Felder existieren und welche Datentypen diese beinhalten dürfen. Gibt aber immer mehr das Schema-less Prinzip: also Import ohne Schema. 
Solr hat zwar eine integrierte Suchoberfläche, die aber nur zu Demo-Zwecken gedacht ist. Technisch können Suchen nachgestellt werden, ist aber wirklich nur zum Testen gedacht. Wenn man also Solr verwenden möchte, muss man selbst noch eine Website machen, oder eine andere Software dazu verwenden,
VuFind basiert auf Solr. 

Suchindex (Solr) oder Datenbank (MySQL): in beide können Daten eingespeist und abgefragt werden.

Solr: Schema kann angesehen werden, dort sieht man welche Felder vorhanden sind. 

In der Konfigurationsdatei von VuFind und Link im BAIN Skript sind die Felder des Solar Index Schema erläutert. 
Haben sich überlegt welche Daten aus MARC in welches Feld soll. Gibt bspw. bei den Titeln verschiedene Felder: 1x Haupttitel, 1x Alternativtitel, 1x Untertitel: wird dann im Ranking anders behandelt, also bspw., wenn Wort im Haupttitel vorkommt wird dieser höher gerankt, als wenn das Wort im Alternativtitel vorkommt. 
Text wird zerlegt. String wird nicht zerlegt: wenn exakt getroffen wird höher gerankt. 

Wertung wird ja eh nur gemacht, wenn man nicht in einem bestimmten Feld sucht
Übung: Bei DSpace und ArchivesSpace Daten kommt eine Fehlermeldung, weil eine ID fehlt.
Die ID ist vom MARC die 001. Wenn man MARC Daten validiert, wird kein Fehler angezeigt, wenn die 001 fehlt, da es nicht Voraussetzung für MARC Daten. Aber in VuFind ist es eine Voraussetzung, deshalb Fehler.
Die 001 kann man im XML einfügen oder mit OpenRefine einfügen. Man könnte auch im Marc.properties ein anderes Feld für die ID festlegen.
